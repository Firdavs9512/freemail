Registration

How to register and use the site:

  1. You will enter the Sing Up section of the site. You can find this section above in the menu section. Or win on the site by clicking on this link.

  2. You can register from the site. The registration system is very convenient and easy:
    • Enter your first and last name
    • Your working email
    • Create a password that only you know
  3. Now you need to create a token to send email. To do this, you will enter the token generation section.
  4. You can put the name of your company or something on the token. This name will later appear in your emails.
  5. You need to choose a suitable template for you. Emails you send will go to the user in the way you choose.

Examples

The token you create has a url that only you can see. You can send email through this.

For Linux users:


curl http://freeservices.uz/api/v1?token={yourToken}&email={to_email}&subject={yourSubject}&message={yourMessage}

You put the token you received from the site in the place where it says yourToken. In the to_email field, write the email of the person to whom you need to send. In the subject field, enter a title for your email. write your message in the message field.

If you use HTML tags, they can change the look of your email!

JavaScript example

How to use JavaScript programming language.

              
const axios = require('axios');

const token = 'yourToken'
const url = 'http://freeservices.uz/api/v1?'

async function sendData(message, email) {
  try {
    const response = await axios.get(url+'token='+token+'&email='email+'&message='+message);
    console.log(response.data);
  } catch (error) {
    console.error(error);
  }
}

sendData('Hello!', 'example@email.com');

Php example

How to use php programming language.


<?php
$url = 'http://freeservices.uz/api/v1?token=';

$data = $url.'yourToken&message='.$yourMessage.'&subject='.$subject.'&email='.$email;

$ch = curl_init();

curl_setopt($ch, CURLOPT_URL, $data);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
$status_code = curl_getinfo($ch, CURLINFO_HTTP_CODE);
curl_close($ch);

if ($status_code == 200) {
    print_r(json_decode($response));
} else {
    echo "Request failed with status code: $status_code\n";
}

?>

Errors

Problems you may encounter. Answers to our most frequently asked questions.

What if the token I bought is not available on the server?
If the token you sent is not available on the server, you will receive an error in JSON format under the name MESSAGE. Check your token to avoid this error.
What if I forgot a parameter when I sent the request?
If you forget a parameter when sending a request, the email will not be sent. You will be sent an error in JSON format. If you forget the subject, it will send your name by default.
What if the token I sent has reached its limit?
If the token you sent does not have a limit, then the mail you sent will not be sent. You will receive an error message in JSON format and we will send you an email that the token limit has been reached.
What if I use HTML tags inside the message?
If you selected an empty template when you created the token, the tags in the message you sent will work and the message you sent will work. If you used a ready-made template, you may have a problem with the style.
I want to change the template in the token?
This is currently not possible. But we can definitely add it in the next update. Now you need to delete the token and create a new one.
Want to hit the primum plan?
We are still working on this. We will definitely fix this in the next update.
